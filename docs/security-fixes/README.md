### 2026-01-22: API Key 硬编码修复 🔴 Critical

**问题**: DeepSeek API key 直接硬编码在源代码中

**影响**: 
- 任何能访问代码仓库的人都能获取 API key
- API key 会被提交到 git 历史记录中
- 可能导致 API 滥用、费用损失或数据泄露

**修复**:
- ✅ 创建配置管理系统
- ✅ 移除源代码中的硬编码
- ✅ 更新 .gitignore
- ✅ 文档脱敏处理
- ✅ 添加安全指南

## 安全修复流程

当发现安全问题时，请遵循以下流程：

### 1. 立即响应
- 评估问题严重性
- 确定影响范围
- 通知相关人员

### 2. 修复实施
- 实施修复方案
- 更新相关代码
- 添加测试验证

### 3. 文档记录
- 创建日期目录
- 编写修复文档
- 更新索引

### 4. 验证发布
- 验证修复效果
- 更新安全指南
- 通知团队成员

### 5. 后续跟进
- 撤销泄露的密钥（如适用）
- 清理 git 历史（如需要）
- 改进预防措施

## 严重性级别

| 级别 | 标识 | 说明 | 响应时间 |
|------|------|------|----------|
| Critical | 🔴 | 严重安全漏洞，需要立即修复 | < 24h |
| High | 🟠 | 高风险问题，需要优先处理 | < 1周 |
| Medium | 🟡 | 中等风险，应及时修复 | < 2周 |
| Low | 🟢 | 低风险，可以计划修复 | < 1月 |
